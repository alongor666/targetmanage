# 🤖 文档协作指引（Agent）

> 面向自动化助手与协作者的文档维护规则与路径索引

## 🎯 文档目标
- 统一 **车险经营目标管理与可视化分析平台** 的业务、架构与实现共识
- 确保设计、架构、业务口径与实现保持一致
- 提供可快速检索、可执行的项目知识库

## 🗂️ 目录结构（按层级）
- `design/`：设计系统与组件规范
- `architecture/`：系统架构、数据模型、组织模式
- `business/`：业务指标、权重、数据校验
- `development/`：开发环境、数据导入、实施规划
- `reference/`：API 参考、故障排除、版本日志

## ✅ 文档更新规则
1. **设计变更** → 更新 `design/`，同步 `design/设计实现指南.md`
2. **架构变更** → 更新 `architecture/`，确认数据接口影响
3. **业务变更** → 更新 `business/`，同步指标/权重口径说明
4. **实施变更** → 更新 `development/`，同步实施计划
5. **变更记录** → 若为版本级更新，补充 `reference/版本更新日志.md`

## 🤝 Codex 协作最佳实践
- 只在 `docs/` 目录内改动，避免影响代码实现
- 先通读 `docs/README.md`，确认文档层级与入口
- 不做无关重构，保持修改范围最小且可追踪
- 文档引用请使用相对路径，保持可点击
- 更新目录或新增文档后，同步文档索引

## 📌 写作规范
- 结论先行，章节标题保持明确可检索
- 使用统一术语与字段命名，避免一词多义
- 与代码实现保持一致，避免“文档漂移”
- 关键公式、口径、接口需给出示例

## 🔍 快速定位建议
- **设计问题** → `design/`
- **数据结构/API** → `architecture/数据模型.md`、`reference/API参考.md`
- **业务计算** → `business/业务指标计算.md`、`business/权重分配规则.md`
- **排错** → `reference/故障排除.md`

## 🧭 推荐维护流程
1. 先查 `docs/README.md` 确认文档层级
2. 定位到目标文档进行修改
3. 同步受影响的上下游文档
4. 如需说明更新，补充版本日志

## 📎 维护提示
- 避免重复解释，优先引用已有文档
- 新增文档需在 `docs/README.md` 增补导航入口
- 保持标题层级一致（`##` → `###` → `####`）

**适用范围**：`docs/` 全目录
