# [P-002] 任务分解模式

> **核心理念**：将复杂任务分解为多个小步骤，让AI逐步完成

**效果提升**：任务完成质量 +35%
**适用场景**：复杂任务、多步骤任务
**难度**：⭐⭐ 中等

---

## 🎯 模式概述

### 问题
❌ 一次性要求完成复杂任务：
```
实现一个完整的用户认证系统
```

结果：AI可能遗漏关键步骤，代码质量参差不齐。

### 解决方案
✅ 分解为步骤，逐个完成：
```
第1步：创建用户数据模型和Zod schema
第2步：实现登录表单组件
第3步：创建认证API路由
第4步：集成状态管理
第5步：添加错误处理
```

---

## 📝 Prompt模板

```markdown
实现[总体目标]

## 任务分解

### Step 1: [子任务1]
- 目标：...
- 输出：...
- 验证：...

### Step 2: [子任务2]
- 目标：...
- 输出：...
- 验证：...

...

## Context
[使用P-001 Context提供模式]

## 执行方式
- 逐步执行，每步完成后等待确认
- 或：一次性返回完整代码（标注步骤）
```

---

## 💡 分解原则

1. **按功能模块分解**：Model → View → Controller
2. **按开发流程分解**：Schema → Logic → UI → Test
3. **按依赖关系分解**：基础 → 核心 → 扩展
