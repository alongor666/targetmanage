## 🚀 场景1：遇到问题时

### 问题示例
你让AI创建一个组件，但AI理解错了你的意图，生成的代码不符合预期。

### 操作流程

#### 步骤1：立即记录（趁热打铁）

```bash
/ai-evolve record
```

**AI会引导你完成**：
1. 问题描述
2. 你的原始Prompt
3. AI的输出
4. 期望 vs 实际
5. 问题分类

#### 步骤2：AI自动分析

Skill会自动：
- 分析问题类型（类型错误/业务逻辑/组件设计/...）
- 识别Prompt的问题（描述不清/上下文不足/...）
- 提供改进建议

#### 步骤3：保存到知识库

自动保存到：
- `docs/ai-evolution/problems/P00X-分类/编号-问题标题.md`
- 更新问题索引

## 🔍 场景2：分析改进

### 问题示例
记录了多个类似的问题后，你想找出根本原因。

### 操作流程

#### 步骤1：触发分析

```bash
/ai-evolve analyze
```

#### 步骤2：AI深度分析

AI会分析：

**技术层面**：
- 代码错误的本质原因
- 技术栈的使用问题

**Prompt层面**：
- Prompt表述是否清晰
- 上下文是否充足
- 任务分解是否合理

**Context层面**：
- 提供的背景信息是否完整
- 业务规则说明是否清楚
- 依赖关系是否明确

#### 步骤3：提炼模式

AI会：
- 提炼通用的Prompt模式
- 设计更好的Context结构
- 生成最佳实践建议

#### 步骤4：保存到解决方案库

自动保存到：
- `docs/ai-evolution/solutions/prompt-patterns/` - Prompt模式
- `docs/ai-evolution/solutions/context-patterns/` - Context模式
- `docs/ai-evolution/solutions/best-practices/` - 最佳实践
- `docs/ai-evolution/analysis/root-causes/` - 根因分析

## 🎓 进阶技巧

### 技巧1：对比学习

**记录成功和失败的Prompt**：
```markdown
## 失败的Prompt ❌
创建一个图表组件

## 成功的Prompt ✅
创建一个ECharts折线图组件，要求：
1. 接收data prop（数组格式）
2. 支持自定义颜色主题
3. 响应式设计，适配移动端
4. 使用TypeScript类型安全
5. 参考项目中UniversalChart的设计模式
```

**分析差异**：
- 失败Prompt：过于简略，缺少具体要求
- 成功Prompt：明确需求、提供上下文、指定参考

**提炼模式**：
→ Prompt模式：**详细需求列表 + 参考现有代码**

### 技巧2：建立个人Prompt库

在 `solutions/prompt-patterns/` 创建你的常用模板：

**创建组件模板**：
```
创建一个[组件类型]组件，要求：
1. 功能：[具体功能描述]
2. Props：[接口定义]
3. 样式：[设计要求]
4. 类型：使用TypeScript
5. 参考：[项目中类似组件]
```

**修复Bug模板**：
```
修复以下问题：
- 文件：[文件路径]
- 错误：[错误信息]
- 复现：[复现步骤]
- 期望：[期望行为]
- 上下文：[相关代码/业务逻辑]
```

### 技巧3：Context提供清单

每次给AI任务前，检查是否提供：

- [ ] **业务背景**：这个功能的业务目的是什么？
- [ ] **技术栈**：使用什么框架、库、工具？
- [ ] **代码规范**：项目的编码规范是什么？
- [ ] **参考代码**：有类似的代码可以参考吗？
- [ ] **依赖关系**：与哪些模块有关联？
- [ ] **约束条件**：有什么限制或要求？

### 技巧4：标签系统

给问题添加标签，便于检索：

```markdown
---
分类: P003-组件设计
标签:
  - TypeScript
  - ECharts
  - 类型推断
  - 泛型
难度: 中等
重要性: 高
状态: 已解决
---
```

## 💡 常见问题

### Q1: 每个问题都要记录吗？
**A**: 不是。推荐记录：
- ✅ 花费时间较长的问题（>15分钟）
- ✅ 重复出现的问题
- ✅ 有代表性的问题
- ❌ 简单的typo、明显的错误

### Q2: 记录问题很花时间怎么办？
**A**: 使用 `/ai-evolve record` skill，AI会引导你快速完成：
- 1分钟：基本信息录入
- 2分钟：AI自动分析
- 1分钟：确认和保存
- **总计4分钟**，但收益是长期的！

### Q3: 如何快速找到相关的解决方案？
**A**: 使用查询功能：
```bash
# 按关键词
/ai-evolve query "组件设计"

# 按分类
查看 problems/P003-组件设计/

# 按标签
查看 problems/index.md 中的标签索引
```

### Q4: Prompt模式太多，记不住怎么办？
**A**: 不需要记住！
1. 使用 `/ai-evolve query` 查询
2. 查看 `solutions/prompt-patterns/index.md` 索引
3. AI会在合适的时候主动推荐

### Q5: 如何衡量自己的进化？
**A**: 使用进化指标：
```bash
/ai-evolve report
```
查看：
- Prompt一次成功率
- 问题解决速度
- 知识复用率
- 模式积累数量
