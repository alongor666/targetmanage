## 💼 业务术语

### 🎯 目标管理

**年度目标 (Annual Target)**
- 定义: 公司在一个财年内设定的总体经营目标
- 范围: 包括保费收入、利润、市场份额等关键指标
- 特点: 战略性、长期性、指导性

**季度目标 (Quarterly Target)**
- 定义: 将年度目标分解到每个季度的阶段性目标
- 用途: 便于进度跟踪和及时调整
- 特点: 可操作性、时效性、灵活性

**月度目标 (Monthly Target)**
- 定义: 将季度目标进一步细化为每月的具体执行目标
- 用途: 日常管理和绩效考核
- 特点: 具体性、可衡量性、短期性

### 📊 业绩分析

**达成率 (Achievement Rate)**
- 公式: (实际值 ÷ 目标值) × 100%
- 用途: 衡量目标完成程度
- 标准: 100%为达标，>100%为超额，<100%为未达标

**增长率 (Growth Rate)**
- 公式: ((当期值 - 上期值) ÷ 上期值) × 100%
- 用途: 衡量业务发展速度
- 类型: 同比增长、环比增长、定基增长

**赔付率 (Loss Ratio)**
- 公式: (理赔支出 ÷ 保费收入) × 100%
- 用途: 衡量承保质量
- 标准: 一般控制在60%-70%之间

**费用率 (Expense Ratio)**
- 公式: (费用支出 ÷ 保费收入) × 100%
- 用途: 衡量运营效率
- 包括: 手续费、管理费用、营销费用等

### 🏢 组织管理

**省级机构 (Provincial Branch)**
- 定义: 在省级行政区域设立的分支机构
- 级别: 二级管理机构
- 职责: 区域内的业务统筹和监督管理

**市级机构 (City Branch)**
- 定义: 在地级市设立的分支机构
- 级别: 三级管理机构
- 职责: 具体业务的执行和管理

**分支机构 (Sub-branch)**
- 定义: 在区县或乡镇设立的业务网点
- 级别: 四级管理机构
- 职责: 直接面向客户的业务办理

## 🛠️ 技术术语

### 🎨 前端技术

**Next.js**
- 定义: 基于 React 的全栈框架
- 特点: SSR/SSG、文件路由、API 路由
- 版本: 14.x

**TypeScript**
- 定义: JavaScript 的超集，添加了类型系统
- 优势: 类型安全、IDE 支持、重构友好
- 版本: 5.x

**Tailwind CSS**
- 定义: 原子化 CSS 框架
- 特点: 实用性优先、高度可定制
- 版本: 3.x

**ECharts**
- 定义: 基于 Canvas 的可视化图表库
- 特点: 功能丰富、性能优秀、配置灵活
- 用途: 数据可视化展示

### 🗄️ 数据管理

**Zustand**
- 定义: 轻量级状态管理库
- 特点: 简单易用、TypeScript 友好
- 用途: 客户端状态管理

**React Query**
- 定义: 服务器状态管理库
- 特点: 缓存、重试、同步
- 用途: API 数据获取和管理

**SWR**
- 定义: 数据获取策略
- 特点: 缓存优先、自动重新验证
- 全称: Stale-While-Revalidate

### 🎯 设计系统

**设计令牌 (Design Tokens)**
- 定义: 设计系统中最小的设计决策单元
- 包括: 颜色、字体、间距、阴影等
- 用途: 保证设计一致性

**原子设计 (Atomic Design)**
- 定义: 从小到大的设计方法论
- 层次: 原子 → 分子 → 有机体 → 模板 → 页面
- 应用: 组件设计和组织

**响应式设计 (Responsive Design)**
- 定义: 适配不同设备和屏幕的设计方法
- 技术: 媒体查询、弹性布局、相对单位
- 断点: sm(640px)、md(768px)、lg(1024px)、xl(1280px)

## 📊 指标术语

### 💰 财务指标

**保费收入 (Premium Income)**
- 定义: 保险公司收取的保险费用总额
- 分类: 原保险保费、再保险保费
- 计算: 签单保费 + 增保保费 - 退保保费

**已赚保费 (Earned Premium)**
- 定义: 在责任期内实际承担风险的保费
- 计算: 保费收入 × (已承担责任期 ÷ 总责任期)
- 用途: 计算赔付率和费用率的基础

**准备金 (Reserves)**
- 定义: 为未来赔付提取的资金准备
- 类型: 未到期责任准备金、赔款准备金
- 原则: 充足性、谨慎性、一致性

### 📈 业务指标

**保单数量 (Policy Count)**
- 定义: 有效保单的总数量
- 分类: 新保单、续保单、终止保单
- 统计: 按产品、渠道、时间维度统计

**保额 (Sum Insured)**
- 定义: 保险合同约定的最高赔付金额
- 用途: 风险衡量和保费计算
- 原则: 保险利益原则

**客户数 (Customer Count)**
- 定义: 购买保险产品客户的总数量
- 分类: 新客户、老客户、活跃客户
- 指标: 客户增长率、客户留存率

### 🎯 绩效指标

**市场份额 (Market Share)**
- 公式: (本公司保费 ÷ 市场总保费) × 100%
- 用途: 衡量市场地位和竞争力
- 类型: 整体市场份额、细分市场份额

**续保率 (Renewal Rate)**
- 公式: (续保保单 ÷ 到期保单) × 100%
- 用途: 衡量客户满意度和忠诚度
- 目标: 一般要求80%以上

**出单率 (Policy Issuance Rate)**
- 公式: (实际出单 ÷ 报价次数) × 100%
- 用途: 衡量销售转化效率
- 影响: 销售策略和产品定价

## 🔄 流程术语

### 📋 业务流程

**目标制定 (Target Setting)**
- 定义: 确定各级组织和个人目标的过程
- 步骤: 分析 → 设定 → 分解 → 审批 → 发布
- 原则: SMART原则、公平性、可实现性

**业绩评估 (Performance Evaluation)**
- 定义: 对目标完成情况进行评价的过程
- 方法: 定量评价、定性评价、综合评价
- 周期: 月度评估、季度评估、年度评估

**偏差分析 (Variance Analysis)**
- 定义: 分析实际与目标差异原因的过程
- 类型: 数量差异、价格差异、效率差异
- 用途: 改进管理和决策制定

### 🛠️ 技术流程

**代码审查 (Code Review)**
- 定义: 对代码质量进行检查和评估的过程
- 内容: 功能正确性、代码规范、性能优化
- 工具: Git Pull Request、自动化检查工具

**持续集成 (Continuous Integration)**
- 定义: 频繁集成代码并自动构建测试的过程
- 目的: 早期发现和解决问题
- 工具: Jenkins、GitHub Actions、GitLab CI

**版本控制 (Version Control)**
- 定义: 管理代码变更历史的系统
- 工具: Git、SVN、Mercurial
- 操作: 提交、分支、合并、回滚

### 📊 数据流程

**数据清洗 (Data Cleansing)**
- 定义: 识别和修正数据错误的过程
- 任务: 去重、补全、格式化、验证
- 目标: 提高数据质量和准确性

**数据聚合 (Data Aggregation)**
- 定义: 将细粒度数据汇总为高层次数据的过程
- 方法: 求和、平均、最大值、最小值
- 维度: 时间、组织、产品、渠道

**数据可视化 (Data Visualization)**
- 定义: 将数据转换为图形形式的过程
- 目的: 便于理解和发现规律
- 工具: ECharts、D3.js、Tableau

## 🔗 相关概念

### 🎯 关联术语

**目标管理 ↔ 绩效管理**
- 目标管理: 设定和分解目标
- 绩效管理: 评估和改进绩效
- 关系: 目标是绩效评估的基础

**业务流程 ↔ 技术实现**
- 业务流程: 业务操作的逻辑顺序
- 技术实现: 业务流程的代码实现
- 关系: 技术服务于业务需求

**数据质量 ↔ 决策质量**
- 数据质量: 数据的准确性和完整性
- 决策质量: 基于数据做出的决策效果
- 关系: 高质量数据支撑高质量决策
